# Security Policy

## Reporting a Vulnerability

We take the security of forge seriously. If you discover a security vulnerability,
please report it privately by emailing security@useforge.dev.

**Do not report security vulnerabilities through public GitHub issues.**

Please include the following in your report:

- Type of vulnerability
- Steps to reproduce
- Affected versions
- Potential impact

We will acknowledge receipt within 48 hours and provide a timeline for
resolution. We will notify you when the vulnerability is fixed and may
request a coordinated disclosure.

## Supported Versions

We provide security patches for the latest minor release. Older versions
should be upgraded as soon as possible.

## Disclosure Policy

When we receive a security report, we will:

1. Confirm receipt within 48 hours
2. Assess the severity and impact
3. Work on a fix
4. Release a patched version
5. Publicly disclose the vulnerability after the fix is available

We believe in coordinated disclosure and will work with reporters to
schedule public disclosure appropriately.
