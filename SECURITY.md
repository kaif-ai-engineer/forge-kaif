# Security Policy

## Supported Versions

We provide security patches for the latest minor release. Older versions
should be upgraded as soon as possible.

| Version | Supported          |
| ------- | ------------------ |
| >= 0.2  | :white_check_mark: |
| < 0.2   | :x:                |

## Reporting a Vulnerability

We take the security of forge seriously. If you discover a security
vulnerability, please report it privately.

**Do not report security vulnerabilities through public GitHub issues.**

### How to report

Send an email to **[EMAIL_ADDRESS]** with the following details:

- Type of vulnerability
- Steps to reproduce (including any proof of concept)
- Affected versions
- Potential impact

You may encrypt sensitive information using our GPG key (available on
request).

### What to expect

1. **Acknowledgment** within 48 hours of your report.
2. **Assessment** of severity and impact.
3. **Fix development** — we will work on a patch and keep you informed
   of progress.
4. **Release** of a patched version.
5. **Public disclosure** after the fix is available, coordinated with
   you.

We believe in coordinated disclosure and will work with you to schedule
public disclosure appropriately.

## Scope

This security policy covers the `forge-kaif` PyPI package and the
`forge` source code in this repository. It does not cover third-party
dependencies or applications built with forge.

## Recognition

We maintain a Hall of Fame for researchers who responsibly disclose
security vulnerabilities. If you would like to be credited, please
indicate your preference when reporting.
